# SPHERE-3 Симуляция на базе Geant4

## Описание

SPHERE-3 G4 — высокопроизводительная симуляция Монте-Карло на базе Geant4 для моделирования распространения фотонов и их детектирования в установке SPHERE-3 для регистрации космических лучей. Симулятор отслеживает черенковские фотоны от широких атмосферных ливней (ШАЛ) через геометрию детектора, моделируя отклики фотоумножителей (ФЭУ) с 2653 пикселями на 379 модулях ФЭУ.

## Быстрый старт

### Сборка и запуск
```bash
# Конфигурация и сборка (Release режим с оптимизациями)
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)

# Запуск симуляции
./SPHERE-3

# Сборка без UI/визуализации (batch режим)
cmake .. -DWITH_GEANT4_UIVIS=OFF
```

### Ключевые файлы
- `SPHERE-3.cpp` - Главный файл симуляции
- `input.ini` - Конфигурационный файл (высота детектора, свойства частиц)
- `phels/` - Директория с входными файлами фотонов
- `vis.mac` - Макрос визуализации

### Конфигурация
Отредактируйте `input.ini` для установки высоты детектора:
```ini
[DEFAULT]
Height = 1000
```

## Полная реализация

### Архитектура

Симуляция следует стандартной архитектуре Geant4 с пользовательскими реализациями:

**Основные компоненты:**
- **DetectorConstruction** (`sphmirrDetectorConstruction.*`) - Определяет геометрию детектора с 379 ФЭУ и 2653 пикселями
- **PhysicsList** (`sphmirrPhysicsList.*`) - Пользовательские физические процессы для распространения оптических фотонов
- **PrimaryGeneratorAction** (`sphmirrPrimaryGeneratorAction.*`) - Читает входные данные фотонов из директории `phels/`
- **EventAction** (`sphmirrEventAction.*`) - Обрабатывает события и вывод данных
- **RunAction** (`sphmirrRunAction.*`) - Управляет инициализацией и завершением сеанса
- **SteppingAction** (`sphmirrSteppingAction.*`) - Отслеживает траектории фотонов и попадания в ФЭУ
- **StackingAction** (`sphmirrStackingAction.*`) - Контролирует приоритеты отслеживания частиц
- **SensitiveDetector** (`sphmirrSD.*`, `sphmirrHit.*`) - Регистрирует попадания фотонов в ФЭУ

**Геометрия детектора:**
- 379 модулей ФЭУ со сферической зеркальной геометрией
- 2653 индивидуальных пикселя с отслеживанием позиции и углов
- Настраиваемая высота детектора (через `input.ini`)
- Углы поворота детектора: phi и theta настраиваются в коде
- Фильтрация временного окна: `tmin=1.8e6 ns`, `tmax=1.8e5 ns`

**Физика:**
- Отслеживание оптических фотонов с черенковским излучением
- Модель чувствительности ФЭУ: `sensitivity(xi) = cos(xi)^1.093`
- Пользовательский список физических процессов для симуляции ШАЛ
- Оптимизации fast-math для производительности

### Оптимизации производительности

CMakeLists.txt включает агрессивные оптимизации компилятора:
- `-O3` - Максимальный уровень оптимизации
- `-march=native` - Векторизация под конкретный CPU
- `-flto` - Оптимизация времени компоновки
- `-ffast-math` - Быстрые операции с плавающей точкой
- `-funroll-loops` - Развертывание циклов
- Поддержка `ccache` для ускорения пересборки

### Поток данных

1. **Вход**: Файлы фотонов из директории `phels/` (формат CORSIKA)
2. **Обработка**: Geant4 отслеживает фотоны через геометрию детектора
3. **Детектирование**: Попадания в ФЭУ регистрируются с временем и позицией
4. **Выход**: Паттерны попаданий записываются в выходные файлы (префикс настраивается)

## Сборка и установка

### Требования

- **Geant4** (10.7 или новее) с UI и драйверами визуализации
- **CMake** 3.16 или новее
- Компилятор с поддержкой **C++20** (GCC 10+, Clang 12+, или MSVC 2019+)
- **Опционально**: ccache для ускорения пересборки

### Шаги установки

1. **Установка Geant4**
   ```bash
   # На macOS (через Homebrew)
   brew install geant4

   # На Ubuntu/Debian
   sudo apt install geant4 geant4-data

   # Или сборка из исходников: https://geant4.web.cern.ch/
   ```

2. **Клонирование и сборка**
   ```bash
   git clone <repository-url>
   cd SPHERE-3_G4
   mkdir build && cd build
   cmake .. -DCMAKE_BUILD_TYPE=Release
   make -j$(nproc)
   ```

3. **Проверка установки**
   ```bash
   # Проверка окружения Geant4
   geant4-config --version

   # Быстрый тест
   ./SPHERE-3
   ```

### Опции сборки

| Опция | Описание | По умолчанию |
|-------|----------|--------------|
| `CMAKE_BUILD_TYPE` | `Release` или `Debug` | `Release` |
| `WITH_GEANT4_UIVIS` | Включить UI/визуализацию | `ON` |
| `CMAKE_CXX_STANDARD` | Стандарт C++ (17/20) | `20` |

## Руководство по использованию

### Базовый процесс симуляции

1. **Подготовка входных файлов**
   - Разместите файлы фотонов в директории `phels/`
   - Файлы должны содержать данные фотонов CORSIKA

2. **Конфигурация симуляции**
   Отредактируйте `input.ini`:
   ```ini
   [DEFAULT]
   Height = 500  # Высота детектора в метрах
   ```

3. **Запуск симуляции**
   ```bash
   cd build
   ./SPHERE-3
   ```
   Симуляция будет:
   - Читать все файлы из директории `phels/`
   - Обрабатывать до 2 миллиардов событий (`NbOfEvents = 2000000000`)
   - Применять фильтрацию временного окна
   - Отслеживать фотоны через детектор

4. **Анализ выходных данных**
   - Данные о попаданиях записываются в выходные файлы (настраивается через `NameOut`)
   - Включает индексы пикселей, время и информацию о позиции

### Режим визуализации

Для запуска с интерактивной визуализацией:

1. **Включение визуализации** (если отключена при сборке)
   ```bash
   cmake .. -DWITH_GEANT4_UIVIS=ON
   make
   ```

2. **Запуск с визуализацией**
   - Раскомментируйте код визуализации в `SPHERE-3.cpp` (строки 122-138)
   - Используйте предоставленные макросы:
   ```bash
   ./SPHERE-3
   # В GUI: /control/execute vis.mac
   ```

3. **Макросы визуализации**
   - `init_vis.mac` - Инициализация визуализации
   - `vis.mac` - Стандартная настройка визуализации

### Пакетная обработка

Для крупномасштабных продакшн расчетов:

1. **Отключение визуализации** (более быстрое выполнение)
   ```bash
   cmake .. -DWITH_GEANT4_UIVIS=OFF -DCMAKE_BUILD_TYPE=Release
   make
   ```

2. **Запуск в фоновом режиме**
   ```bash
   nohup ./SPHERE-3 > simulation.log 2>&1 &
   ```

3. **Мониторинг прогресса**
   ```bash
   tail -f simulation.log
   ```

### Параметры конфигурации

**Жестко заданные параметры** (в `SPHERE-3.cpp`):
- `zstart` - Начальная высота фотона
- `phi`, `the` - Углы поворота детектора (по умолчанию: 0°)
- `tmin`, `tmax` - Временное окно для детектирования попаданий
- `NbOfEvents` - Максимальное количество событий для обработки (2×10⁹)
- `myseed` - Seed генератора случайных чисел (3453544)

**Конфигурация во время выполнения** (`input.ini`):
- `Height` - Высота детектора

Для изменения других параметров отредактируйте глобальные переменные в `SPHERE-3.cpp` и перекомпилируйте.

## Устранение неполадок

### Проблемы сборки

**Проблема**: `Geant4 package not found`
```
CMake Error: find_package(Geant4) failed
```
**Решение**:
- Убедитесь, что Geant4 установлен: `geant4-config --version`
- Установите переменную окружения: `export G4INSTALL=/path/to/geant4`
- Загрузите окружение Geant4: `source /path/to/geant4/bin/geant4.sh`

---

**Проблема**: Ошибки компилятора C++20
```
error: 'filesystem' is not a namespace-name
```
**Решение**:
- Обновите компилятор: GCC 10+, Clang 12+, или MSVC 2019+
- Проверьте, что CMake находит правильный компилятор:
  ```bash
  cmake .. -DCMAKE_CXX_COMPILER=g++-11
  ```

---

**Проблема**: Ошибки компоновки с библиотеками Geant4
**Решение**:
- Проверьте, что установка Geant4 полная (включая файлы данных)
- Проверьте вывод CMake на отсутствующие компоненты Geant4
- Пересоберите Geant4 с соответствующей конфигурацией

### Проблемы во время выполнения

**Проблема**: `No input files found in phels/`
**Решение**:
- Проверьте, что директория `phels/` существует в рабочей директории
- Проверьте, что переменная `CurrentPath` в `SPHERE-3.cpp` соответствует вашей системе
- Обновите жестко заданные пути:
  ```cpp
  CurrentPath = "/path/to/your/SPHERE-3_G4/";
  ```

---

**Проблема**: Симуляция запускается, но не производит вывод
**Решение**:
- Проверьте параметры временного окна (`tmin`, `tmax`) на разумность
- Проверьте, что входные файлы фотонов имеют правильный формат
- Включите подробный вывод для отладки трассировки
- Проверьте дисковое пространство для выходных файлов

---

**Проблема**: Segmentation fault при инициализации
**Решение**:
- Проверьте конструкцию геометрии детектора
- Проверьте, что массивы координат ФЭУ правильно инициализированы
- Запустите в режиме отладки с gdb:
  ```bash
  gdb ./SPHERE-3
  (gdb) run
  (gdb) backtrace
  ```

---

**Проблема**: Проблемы производительности (медленное выполнение)
**Решение**:
- Убедитесь, что собрано в режиме Release: `cmake .. -DCMAKE_BUILD_TYPE=Release`
- Включите оптимизации компилятора (уже в CMakeLists.txt)
- Отключите визуализацию для продакшн расчетов
- Проверьте системные ресурсы (CPU, память)

### Проблемы с данными

**Проблема**: Неправильные паттерны попаданий
**Решение**:
- Проверьте, что формат входных файлов фотонов соответствует ожидаемой структуре CORSIKA
- Проверьте параметры геометрии детектора (позиции ФЭУ, координаты пикселей)
- Проверьте настройки временного окна для вашей геометрии ливня
- Просмотрите конфигурацию списка физических процессов

---

**Проблема**: Отсутствующие события или низкая статистика
**Решение**:
- Увеличьте `NbOfEvents` в `SPHERE-3.cpp`
- Проверьте логику чтения файлов в `PrimaryGeneratorAction`
- Проверьте, что все файлы фотонов обрабатываются
- Просмотрите приоритеты stacking action
